## Components

Ordered by how often they're mis-used.

### Button
- **For:** any clickable action — submit, save, delete, open a dialog. Not navigation.
- **Use instead of:** a raw `<button className="bg-indigo-600 ...">` or a
  `<div onClick>`. Hand-rolled action buttons are the most common violation.
- **Import:** `@/components/ui/button`
- **Key props / variants:** `variant="primary|secondary|ghost|destructive"`
  (default `primary`), `size="sm|md|lg"` (default `md`). Destructive actions →
  `variant="destructive"`. Icon-only → must pass `aria-label`.
- **Notes:** for navigation use `TextLink`, not a Button.

### TextLink
- **For:** navigation — going somewhere.
- **Use instead of:** a raw `<a>` styled to look like a link, or a Button used
  for nav.
- **Import:** `@/components/ui/text-link`

### Card
- **For:** a surface container grouping one block of content.
- **Use instead of:** a hand-rolled `<div className="border rounded shadow">`
  surface.
- **Import:** `@/components/ui/card` (`Card`, `CardHeader`, `CardTitle`, `CardContent`)
- **Notes:** **never nest a Card inside a Card.** The inner thing should be a
  plain section with spacing, not another surface.

### Input / Field
- **For:** form inputs.
- **Use instead of:** a bare `<input>` with no label.
- **Import:** `@/components/ui/input`, `@/components/ui/field`
- **Notes:** always wrap an Input in `<Field>` so it gets a label + error slot.

### Modal
- **For:** dialogs.
- **Use instead of:** the deprecated `LegacyModal` — do not use it in new code.
- **Import:** `@/components/ui/modal`

## Tokens / Scales

### Color
| Token | Value | Use for |
|-------|-------|---------|
| `--color-primary` | `#4f46e5` | the one primary action per region |
| `--color-destructive` | `#dc2626` | destructive actions only |
| `--color-text` | `#18181b` | body text |
| `--color-text-muted` | `#71717a` | secondary text |
| `--color-border` | `#e4e4e7` | borders (use sparingly — see Principles) |
- **Reference as:** CSS vars (`var(--color-primary)`) or the mapped theme class.
- **Never:** raw hex in components; indigo or red for decoration (they're
  reserved for primary / destructive actions).

### Spacing
- **Scale:** `space-1=4px, space-2=8px, space-3=12px, space-4=16px,
  space-6=24px, space-8=32px`. This is the complete allowed set.
- **Reference as:** the `space-*` tokens / mapped Tailwind `p-*`/`gap-*`/`m-*`.
- **Never:** arbitrary px (e.g. `20px`, `10px`) outside the scale.

### Typography
- **Type scale:** `text-sm=13px, text-base=15px, text-lg=18px, text-xl=24px`.
- **Weights allowed:** `400` (normal) and `600` (semibold) **only**.
- **Never:** off-scale sizes; weights like `500` or `700`.

### Radius
- **Allowed:** `radius-sm=6px, radius-md=8px, radius-lg=12px`.

## Principles

Distilled from inputs/brand.md. Each is falsifiable — you can point at a screen
and say it's violated.

- **Restraint over decoration.** No gradients, no flourish shadows. Never more
  than two font weights (400 / 600) on a screen. If a screen feels busy, remove
  something.
- **Whitespace separates; borders are a last resort.** Group with space, not
  dividers. A wall of bordered boxes reads as cluttered. Prefer generous padding.
- **One primary action per view.** Exactly one indigo `variant="primary"` button
  per screen region. Everything else is secondary or ghost. Two competing
  primaries is a bug.
- **Muted by default, color with intent.** Indigo = primary action only. Red =
  destructive only. No brand color for decoration.

## Anti-patterns

- Hand-rolled action buttons (`<button className="bg-indigo-600 ...">`) instead
  of `Button`.
- Nested Cards (a Card inside a Card).
- Bare `<input>` with no `<Field>` / label.
- Use of the deprecated `LegacyModal` in new code.
- Two primary buttons competing in one view.
- Off-scale spacing (raw px not in the `space-*` set).
